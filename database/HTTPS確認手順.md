# HTTPS 確認手順

## 1. アプリケーションのURLを確認

ブラウザでアプリケーションにアクセスし、アドレスバーのURLを確認してください。

### ✅ 正しい例（安全）
```
https://your-domain.com/
```
- URLが `https://` で始まっている
- アドレスバーに🔒（鍵マーク）が表示されている

### ❌ 間違った例（危険）
```
http://your-domain.com/
```
- URLが `http://` で始まっている
- 🔒マークがない

---

## 2. 証明書の確認（詳細）

### Chrome/Edge の場合
1. アドレスバーの 🔒 マークをクリック
2. 「証明書」または「接続は保護されています」をクリック
3. 以下を確認：
   - 発行者: Let's Encrypt, Google Trust Services など
   - 有効期限: 期限切れでないこと

### Firefox の場合
1. アドレスバーの 🔒 マークをクリック
2. 「接続は安全です」→「詳細を表示」
3. 「証明書を表示」をクリック

---

## 3. デプロイ環境別のHTTPS設定

### Vercel / Netlify / GitHub Pages
- **自動でHTTPS有効化** ✅
- カスタムドメインを設定しても自動で証明書発行

### AWS / GCP / Azure
- 手動でSSL証明書の設定が必要
- Let's Encrypt で無料証明書を取得可能

### ローカル環境（開発用）
```
http://localhost:3000  ← これは問題なし（ローカルのみ）
```
- ローカル環境は `http://` でも問題ありません
- **本番環境では必ず `https://` を使用してください**

---

## 4. HTTPSが無効の場合のリスク

### 🚨 発生する危険
1. **通信内容の盗聴**
   - Wi-Fiで通信内容が見られる
   - パスワードが平文で送信される

2. **中間者攻撃（MITM）**
   - 攻撃者が通信を改ざんできる
   - 偽のログイン画面に誘導される

3. **個人情報保護法違反**
   - 個人情報を暗号化せずに送信

---

## 5. HTTPSを有効化する方法

### 最も簡単な方法: Vercel にデプロイ

1. Vercel アカウント作成（無料）
   https://vercel.com/

2. GitHubリポジトリを接続

3. デプロイ
   - 自動でHTTPSが有効になります
   - カスタムドメインも無料でHTTPS化

### その他の方法
- **Netlify**: Vercel と同様に簡単
- **Cloudflare Pages**: 無料でHTTPS + CDN
- **Firebase Hosting**: Google提供、無料プラン有り

---

## 6. 確認完了チェックリスト

- [ ] URLが `https://` で始まっている
- [ ] ブラウザに🔒マークが表示されている
- [ ] 証明書の有効期限が切れていない
- [ ] ログイン時にブラウザの警告が出ない
- [ ] すべてのページでHTTPSになっている（一部だけHTTPSではダメ）

---

## トラブルシューティング

### Q1: 「この接続ではプライバシーが保護されません」と表示される
A: SSL証明書が無効または期限切れです。証明書を更新してください。

### Q2: 一部のページだけHTTPSになっている
A: Mixed Content エラーの可能性があります。すべてのリソース（画像、CSS、JSなど）もHTTPSで読み込む必要があります。

### Q3: ローカル環境でHTTPSにする必要は？
A: 開発環境では不要です。本番環境のみHTTPS必須です。

---

## 次のステップ

✅ HTTPSが有効 → 問題なし
❌ HTTPSが無効 → Vercel等にデプロイしてHTTPSを有効化
