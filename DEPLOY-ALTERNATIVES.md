# プライベートリポジトリから無料ホスティングする方法

## 🎯 GitHub Pagesの制限

GitHub Pagesは無料プランでは**パブリックリポジトリのみ**対応しています。
プライベートリポジトリでGitHub Pagesを使うには、**GitHub Pro（$4/月）以上**が必要です。

## ✅ 無料で使える代替案（プライベートリポジトリ対応）

### 1. Vercel（最も推奨・最も簡単）⭐

**メリット：**
- ✅ 完全無料（個人利用）
- ✅ プライベートリポジトリ対応
- ✅ HTTPS自動対応
- ✅ 自動デプロイ（GitHubと連携）
- ✅ カスタムドメイン対応
- ✅ CDNによる高速配信

**デプロイ手順：**
1. https://vercel.com にアクセス
2. 「Sign Up」→ GitHubアカウントでログイン
3. 「Add New Project」をクリック
4. プライベートリポジトリを選択
5. 設定はそのまま「Deploy」をクリック
6. 完了！無料のHTTPS URLが発行されます

**所要時間：約3分**

---

### 2. Netlify（Vercelと同等）

**メリット：**
- ✅ 完全無料
- ✅ プライベートリポジトリ対応
- ✅ HTTPS自動対応
- ✅ 自動デプロイ

**デプロイ手順：**
1. https://netlify.com にアクセス
2. 「Sign up」→ GitHubアカウントでログイン
3. 「Add new site」→ 「Import an existing project」
4. プライベートリポジトリを選択
5. 設定はそのまま「Deploy site」をクリック

---

### 3. Cloudflare Pages（無料で高機能）

**メリット：**
- ✅ 完全無料
- ✅ プライベートリポジトリ対応
- ✅ Cloudflare CDN（非常に高速）
- ✅ 無制限の帯域幅

**デプロイ手順：**
1. https://pages.cloudflare.com にアクセス
2. GitHubアカウントでログイン
3. 「Create a project」→ リポジトリを選択
4. 「Save and Deploy」をクリック

---

### 4. Render（サーバーレス対応）

**メリット：**
- ✅ 無料プランあり
- ✅ プライベートリポジトリ対応
- ✅ HTTPS自動対応

## ⚠️ 重要な注意点

### フロントエンドのみの場合（静的サイト）

`index.html` をそのままホスティングする場合は、Vercel/Netlifyが最適です。

### バックエンドサーバー（server.js）が必要な場合

現在のコードには `server.js` があるようですが、実際に使っているか確認が必要です。

- **フロントエンドのみ** → Vercel/Netlify/Cloudflare Pages
- **バックエンド必要** → Render/Railway/Heroku（無料プランあり）

## 🚀 すぐに始める（Vercel編）

1. https://vercel.com/signup にアクセス
2. GitHubアカウントでサインアップ
3. 「New Project」→ プライベートリポジトリを選択
4. 「Deploy」をクリック
5. 完了！

**URL例：** `https://your-project.vercel.app`

このURLをチームメンバーに共有するだけで、誰でもアクセスできます（コードは非公開のまま）。
