# StockEasy | 備品管理アプリ

特別養護老人ホームの相談員4名で共有して使う備品管理Webアプリです。
Google Apps Script（GAS）+ Googleスプレッドシートで動作します。

- 認証機能なし（URLを知っている人だけが使う運用）
- 借用・返却・履歴閲覧：ゲスト含む全員が利用可能
- 備品・職員・カテゴリの管理：管理者のみ（管理者ID + パスワードでログイン）

---

## 1. ファイル構成

| ファイル | 内容 |
|---|---|
| `Code.gs` | サーバーサイド（API・スプレッドシート読み書き・初期セットアップ） |
| `index.html` | フロントエンド（全画面・CSS・JSを含む単一ファイル） |

---

## 2. スプレッドシートのセットアップ手順

**シートは手動で作成する必要はありません。** セットアップ関数が自動で作成します。

1. [Google スプレッドシート](https://sheets.google.com) で新規スプレッドシートを作成する
   （名前は任意。例：`StockEasy データ`）
2. メニューの **拡張機能 > Apps Script** を開く
3. Apps Script エディタで：
   - 既定の `コード.gs` の中身をすべて削除し、本リポジトリの `Code.gs` の内容を貼り付ける
   - **ファイル > ＋ > HTML** で `index` という名前のファイルを作成し、`index.html` の内容を貼り付ける
     （ファイル名は拡張子なしの **`index`** にすること）
4. プロジェクト名を「StockEasy」などに変更して保存（Ctrl+S）

### 自動作成されるシート

`setupSheets()` 実行後（手順3参照）、以下の5シートが作成されます。

| シート名 | 内容 |
|---|---|
| `items` | 備品マスタ（備品ID / 備品名 / カテゴリ / 状態 / 現在の借用者 / 備考 / 登録日時） |
| `history` | 借用履歴（履歴ID / 備品ID / 備品名 / 借用者名 / 借用日時 / 返却日時 / 操作者） |
| `staff` | 職員マスタ（職員ID / 職員名 / 登録日時） |
| `categories` | カテゴリマスタ（カテゴリID / カテゴリ名 / 登録日時） |
| `admins` | 管理者マスタ（管理者ID / パスワード） |

> 手動で作る場合は、上記のシート名（半角小文字）で作成し、1行目に各見出しを入れてください。

---

## 3. 初期データの投入方法

1. Apps Script エディタ上部の関数選択ドロップダウンで **`setupSheets`** を選ぶ
2. **実行** をクリック
3. 初回は承認画面が出るので、自分のGoogleアカウントで許可する
   （「このアプリは確認されていません」と出た場合は「詳細」→「（プロジェクト名）に移動」で続行）
4. スプレッドシートに5つのシートと以下の初期データが投入されたことを確認する

投入される初期データ：

| 項目 | 値 |
|---|---|
| 管理者 | ID: `admin` / パスワード: `1234` |
| カテゴリ | 移動用具、医療機器、IT機器、日用品 |
| 職員 | （空。設定画面から追加してください） |

`setupSheets()` は何度実行しても既存データを壊しません（不足分のみ補います）。

---

## 4. GASデプロイ手順（Webアプリとして公開）

1. Apps Script エディタ右上の **デプロイ > 新しいデプロイ** をクリック
2. 歯車アイコンから種類で **「ウェブアプリ」** を選択
3. 以下のとおり設定する

   | 設定項目 | 値 |
   |---|---|
   | 説明 | 任意（例：StockEasy v1.0.0） |
   | 次のユーザーとして実行 | **自分**（スプレッドシートのオーナー） |
   | アクセスできるユーザー | **全員**（匿名ユーザーを含む） |

4. **デプロイ** をクリックし、表示された **ウェブアプリのURL**（`https://script.google.com/macros/s/.../exec`）を控える
5. このURLを相談員4名に共有する（URLを知っている人だけが使えます）

### コード修正後の再デプロイ

コードを修正したら **デプロイ > デプロイを管理 > 鉛筆アイコン > バージョン「新バージョン」> デプロイ** で更新します。
（URLは変わりません。「新しいデプロイ」で作り直すとURLが変わるので注意）

---

## 5. 使い方

1. 共有されたURLを開くとログイン画面が表示される
2. 管理者は `admin` / `1234` でログイン（**ログイン後すぐに設定画面 > 管理者設定 でパスワードを変更してください**）
3. 一般利用者は「ゲストとして利用する」をクリック
4. まず管理者が **設定 > 職員管理** で職員を登録し、**設定 > 備品管理** で備品を登録する
5. ダッシュボードから借用・返却・要修理の切り替えを行う

ログイン状態は sessionStorage で管理しています（ページ更新では維持、タブを閉じるとリセット）。

---

## 6. 技術メモ

- 画面は `HtmlService` で配信し、画面からの操作は `google.script.run` 経由で `api()` 関数を呼び出します
- 外部からJSONで読み書きしたい場合のAPIも用意しています
  - 読み取り：`GET {WebアプリURL}?action=getData` / `?action=getHistory`
  - 書き込み：`POST {WebアプリURL}` に `{"action":"borrow","itemId":"ITEM-001","staffName":"山田 太郎","operator":"ゲスト"}` のようなJSONを送信
  - 管理者専用アクション（`addItem` など）は `adminId` / `password` をパラメータに含める必要があります
- すべての書き込みは `LockService`（ScriptLock）で排他制御しており、4名同時操作でも競合しません
- 貸出中の備品への重複借用、返却済み備品への返却、貸出中備品の削除はサーバー側でブロックされます
